第24章 Protective Put 进阶：保险、成本与行权价选择

本章主要基于以下图片材料整理并扩写：

  • renamed/img18.jpg：Protective Put 的应用与行权价选择，强调“持有正股 + 买 Put = 对冲”、ATM / OTM 的保护取舍、开仓时机与 IV 成本，以及用结构组合降低保险成本。

同时参考了权威教育资料中的基础定义：

  • 投资者教育资料通常把 Protective Put 定义为：在持有正股的同时，买入一张 Put，为持股建立下方保护。

如果你只记一句话，那就是：

Protective Put 不是为了让你赚更多，而是为了让你在不卖掉股票的前提下，先把“最坏会坏到哪里”写出来。

很多人到了真正持股阶段，才会第一次认真理解期权的价值。

因为当你已经持有一只股票，而且它：

  • 是你的核心仓位；
  • 你长期依然看好；
  • 但短期又担心财报、消息面、回撤或系统性风险；

这时最痛苦的地方在于：

  • 你不想卖；
  • 可你也不想眼看着利润大幅回吐，甚至把浮盈重新打回原形。

这正是 Protective Put 要解决的问题。

它不是让你变得更激进，而是让你在继续持有的同时，给仓位加上一层明确的下限。

阅读导航：第一次读先抓“保险逻辑”，第二次读再补成本取舍

这一章第一次读，不要先去算最精细的最佳行权价。

你先抓一句：

Protective Put 的任务不是提高收益，而是先把“最坏会坏到哪里”写出来。

第一遍你只要看懂：

  • 它为什么像保险；
  • ATM / OTM Put 的主要差别是什么；
  • 为什么买保护时不能只看股价、不看 IV。

第二遍再去补：

  • 成本怎么平衡；
  • 什么时候 Collar 会成为自然替代；
  • 什么场景下更值得把保护做厚、而不是做便宜。

一、先把 Protective Put 讲成人话：它就是给持股买保险

24.1 结构很简单：持有 100 股正股 + 买一张 Put

Protective Put 的结构并不复杂：

  • 你先持有正股；
  • 然后买入一张 Put；
  • Put 的标的是同一只股票；
  • 通常一张股票期权对应 100 股，所以最常见的配比就是：100 股正股 + 1 张 Put。

这就是图片里那句最核心的话：

持有 100 股正股 + 买 Put = 对冲。

为什么叫“保护性看跌期权”？

因为一旦股价跌破某个区间，Put 会开始增值。

这意味着：

  • 正股在跌；
  • 但 Put 在补；
  • 仓位整体的下跌速度会被明显减缓；
  • 到了某个位置以后，你甚至能把最终损失锁在一个可计算的范围里。

从实战感受上说，它最像的不是投机，而是：

  • 给你的持股买了一份保险。

24.2 它解决的是“我不想卖，但我也不想裸着扛”

Protective Put 特别适合这些情形：

  • 你长期仍然看多；
  • 但短期担心明显回撤；
  • 或者你已经有很大浮盈，不想让利润回吐太多；
  • 或者某只股票在你组合里占比太高，短期需要降一下尾部风险。

这和直接卖股不一样。

直接卖股是：

  • 先退出风险；
  • 同时也退出未来上涨。

Protective Put 则是：

  • 你继续留在场内；
  • 上涨你还能参与；
  • 下跌则不再完全裸奔。

所以它更适合那些：

长期观点没变，但短期不确定性明显升高的时刻。

二、Protective Put 的优点很清楚，但它最真实的缺点也必须讲透

24.3 最大优点：保留上涨，同时把下方风险封住一部分

图片里把它的优点总结得非常准确：

  • 保留股票上涨收益；
  • 消除或显著削弱下方风险；
  • 相当于给自己的持仓买保险。

这意味着，如果股票继续上涨：

  • 你的正股还在；
  • 你不会因为提前卖掉而完全错过行情；
  • 只不过 Put 这部分保险费，大概率会慢慢损耗。

如果股票开始下跌：

  • Put 会逐步开始工作；
  • 你不再是“纯正股暴露”；
  • 而是拥有一个更可控的收益曲线。

这种结构最大的价值，不是让收益更高，而是让你：

  • 更敢继续拿着好公司；
  • 更不容易被短期波动逼出局；
  • 在回撤面前有明确预案，而不是临盘慌乱。

24.4 最大缺点：保险不是免费的，Protective Put 的核心问题就是成本

图片里也毫不含糊地指出：

  • 缺点：成本高。

这一点一定要直说。

Protective Put 最大的难点，从来不是你听不懂结构，而是：

当你真正去买保险时，往往会发现保险很贵。

为什么？

因为你买的是 Put，而 Put 的价格会受到：

  • 剩余期限；
  • 行权价位置；
  • 市场对风险的担心程度；
  • 也就是 IV 水平；

的共同影响。

如果市场已经很慌、IV 已经很高，你才突然想起来要买保护，成本通常不会好看。

这和现实生活买保险一样：

  • 风暴来了才去买，价格往往不便宜。

所以 Protective Put 的关键，不只是“要不要买”，更是：

  • 什么时候买；
  • 买哪一档；
  • 这份保险到底值得你付多高的保费。

三、ATM Put 和 OTM Put，到底该怎么选

24.5 行权价选择，本质上是在选“保护力度”和“保险费”

图片里把这个问题拆得很清楚：

  • ATM Put：保护更直接、更快，但成本更高；
  • OTM Put：更便宜，但要等股价先跌一段，它才会更明显地发挥作用。

这其实就是保险世界最经典的权衡：

  • 你是要更强的保障？
  • 还是要更低的保费？

ATM Put 的特点

  • 更接近当前股价；
  • 一旦下跌，保护更快“接上”；
  • 仓位的下方曲线更早被托住；
  • 代价是权利金更贵。

OTM Put 的特点

  • 离当前股价更远；
  • 价格更便宜；
  • 适合“我担心大跌，但不想为每一小段波动都付太多钱”；
  • 代价是，在到达那条保护线之前，你仍要先承受一段正股回撤。

所以行权价选择，并不是对错题，而是偏好题：

你想买的是“立刻开始保护”的保险，还是“只防大伤”的保险。

24.6 如果你非常确定短期风险很大，ATM 往往更合理

图片里给了一个非常实战的判断框架：

  • 如果你对短期下跌的确定性更强，倾向 ATM Put；
  • 因为你要的是更直接的保护，不想等跌深了才起作用。

这种情境通常包括：

  • 重大事件前夜；
  • 你预期会有剧烈下行波动；
  • 或者某个仓位已经很大，短期不能再承受明显回撤。

这时候，买便宜的 OTM Put 可能会出现一个问题：

  • 你以为自己做了保护；
  • 结果真正开始大跌时，前面那一段损失仍然只能自己扛。

所以如果你的目标是“我现在就想明显削弱回撤”，ATM 往往更贴近这个任务。

24.7 如果你只是想防尾部风险，OTM Put 往往更务实

但不是所有保护，都值得用最贵的保险。

如果你的判断更像这样：

  • 我不是特别确定马上会大跌；
  • 但我担心黑天鹅、财报炸雷、系统性回撤；
  • 或者我只是希望给组合加一层下方底线；

那 OTM Put 常常更务实。

因为它解决的不是“每跌一点我都不想难受”，而是：

  • 我愿意承受正常波动；
  • 但不想在真正大跌时毫无保护。

这也是很多长期持股者更常见的思路：

  • 平时不买最贵的近身保险；
  • 但在关键阶段，用相对更便宜的 OTM Put 防住尾部。

四、Protective Put 最常被忽略的一点：开仓时机往往比图形更重要

24.8 买保险要看 IV，不只是看 K 线

图片里有一句非常值得长期记住的话：

降低成本的关键，不是只盯 K 线，而是看期权的 IV。

这句话很重要，因为很多人做 Protective Put 时，只会这样想：

  • 最近涨很多了；
  • 感觉该跌了；
  • 赶紧买个 Put 保一下。

但如果这时候市场也在高度紧张，IV 已经抬得很高，你买进去的保险通常会非常贵。

所以 Protective Put 的时机，不应该只由股价图形决定，还要加上一个期权维度：

  • 我现在买的保险，是不是已经被市场定得很贵了？

如果 IV 很低时就提前布好保护：

  • Put 往往更便宜；
  • 长期来看，保险性价比也更高。

如果 IV 已经很高：

  • 你依然可以为了风险管理去买；
  • 但必须承认，这时你买的是一份昂贵保险。

这会直接影响后面的行权价和结构选择。

五、保险太贵时，怎么降低成本，但又不把保护做坏

24.9 最直接的办法：接受“不是所有下跌都要全保”

很多 Protective Put 成本高，不是因为这个策略天然无解，而是因为使用者默认想要：

  • 立刻保护；
  • 高强度保护；
  • 持续时间还长；
  • 最好又便宜。

现实里，这四件事往往不能同时满足。

所以第一种降成本方法，不是先上复杂结构，而是先想清楚：

  • 我到底要防哪一段？
  • 是防 3% 的小回撤，还是防 15% 的事故？
  • 是防未来两周，还是未来三个月？

一旦这个问题想清楚，很多不必要的保费，本身就能省掉。

24.10 结构化降成本：把纯保险改造成“有边界的保险”

图片里给出的思路，是在保险太贵时，用结构组合去降低成本。

最常见、也最值得新手先理解的有两种。

方法一：把纯 long put 进一步做成有下限的 put spread 式保护

如果你：

  • 持有正股；
  • 买入一张 Put；
  • 同时再卖出一张更低行权价的 Put；

那本质上就是把原本无限向下延伸的单一保险，改成：

  • 上面先由 long put 提供保护；
  • 但在更深位置，把一部分极端保护让出去，换回一笔权利金。

这样做的好处是：

  • 保险费下降；
  • 比纯 Protective Put 更省钱。

代价是：

  • 保护不再无限延伸；
  • 真正极端的大跌里，你让出了一部分更深层保护。

所以它更适合：

  • 你担心的是一段可预见回撤；
  • 而不是想完全对冲所有尾部风险。

方法二：Protective Put + 卖 Call，也就是 Collar

图片里也把这个方法标得很清楚：

  • Protective Put + 卖 Call = Collar

这个结构你在前面的 Collar 章节已经见过。

它的逻辑是：

  • 用卖 Call 收进的权利金；
  • 去补贴买 Put 的成本；
  • 于是保险更便宜，甚至有机会接近零成本。

代价也非常清楚：

  • 你为了降低下方保护成本；
  • 把一部分上方收益让了出去。

所以 Collar 常常适合这些情形：

  • 你更在意先把下方守住；
  • 对上方继续大幅冲刺没有那么执着；
  • 或者你本来就愿意在某个价位附近逐步止盈。

如果你还想保留巨大的上行想象空间，Collar 就未必是最舒服的结构。

六、Protective Put 最适合哪些真实情境

24.11 重大事件前的短期保险

这是最直观的一类。

比如你持有 NVDA、AAPL、TSLA 这类高关注度股票，而：

  • 财报就在眼前；
  • 指引、监管、宏观消息可能引发大波动；
  • 你又不想在事件前卖掉核心仓位。

Protective Put 就很适合这种：

  • 想继续拿仓位；
  • 又不想裸着过事件。

24.12 已有较大浮盈，不想利润大幅回吐

还有一种非常典型的场景：

  • 你已经持有很久；
  • 仓位浮盈很厚；
  • 你长期还看好；
  • 但短期明显感到波动上升，不想回吐太多。

这时很多人不愿意直接卖股，原因可能包括：

  • 长期 thesis 还在；
  • 税务原因；
  • 组合位置重要；
  • 或者卖了以后再追回来很痛苦。

Protective Put 在这里的角色，不是去提高收益，而是：

让你有机会在不破坏长期仓位的前提下，给已有利润加一道闸门。

24.13 单一个股仓位过大，需要临时降一下尾部风险

如果某只股票在组合里占比过大，哪怕你长期看多，它也会带来一个现实问题：

  • 一旦短期出事，整个组合波动都会明显放大。

这时你未必要立刻削仓。

另一种做法是：

  • 给这部分仓位买一段时间的 Put；
  • 把组合最担心的那一段风险先压下去；
  • 等事件过去或波动消化后，再决定是否继续保留保险。

这也是 Protective Put 非常像“仓位管理工具”的地方。

七、新手最容易犯的五个 Protective Put 误会

24.14 误会一：Protective Put 既然能防跌，那就应该一直买

不对。

保险不是免费的。

如果你长期、无差别、无节制地给所有仓位持续买保险，时间一长，保费本身就可能严重拖累收益。

所以 Protective Put 应该是：

  • 在特定风险阶段使用；
  • 而不是默认永久打开。

24.15 误会二：只要买了 Put，就等于完全没风险

也不对。

你仍然有：

  • Put 本身的成本；
  • 保险失效后的再决策问题；
  • 行权价以上那一段正常波动；
  • 以及买太贵、买太短、买太远的执行问题。

Protective Put 不是让风险消失，而是让风险变得更明确、更可计算。

24.16 误会三：保护一定买 ATM 才专业

不是。

ATM 只是保护更强、更快，但它不一定总是最划算。

如果你的目标只是防尾部，而不是防每一段回撤，OTM Put 往往更符合任务。

24.17 误会四：股价图形看起来危险，就立刻买 Put，不用管 IV

这也是很常见的坑。

如果市场已经极度紧张，Put 的价格往往早就被抬上去了。

你这时买保护，不一定错； 但如果不看 IV，就很容易在“最想省钱的时候买了最贵的保险”。

24.18 误会五：Protective Put 的目的，是提高收益率

这恰恰把它理解反了。

Protective Put 的目的，是：

  • 降低尾部风险；
  • 保留上涨参与；
  • 帮你更安心地继续持有。

它首先是风险管理工具，其次才谈收益效率。

八、把这一章收住：Protective Put 是写给“还想继续持有，但不想再裸着扛”的人

24.19 本章你最该带走的六句话

1. Protective Put = 持有正股 + 买入 Put，本质上是在给持股买保险。 2. 它的最大价值，是保留上涨参与，同时给下方风险建立明确边界。 3. 它最大的难点，不是结构，而是成本。 4. ATM Put 保护更强更快，OTM Put 更便宜但要先承受一段回撤。 5. 买保护时不能只看股价图形，还要看 IV 是否已经太贵。 6. 如果纯 Protective Put 成本过高，可以考虑更有边界的 put spread 保护，或用 Collar 降低保费。

如果说 Long Put 更像是独立的看空仓位， 那 Protective Put 更像是：

  • 你已经拥有一项资产；
  • 你不想失去它；
  • 但你也不想继续毫无遮挡地承受短期风暴。

它不华丽，却非常成熟。

九、自测题

1. 为什么说 Protective Put 更像保险，而不是投机？ 2. 如果你长期看多但短期担心回撤，为什么 Protective Put 比直接卖股更适合某些场景？ 3. ATM Put 和 OTM Put 的核心区别，为什么本质上是“保护力度”和“成本”的取舍？ 4. 为什么买 Protective Put 时，只看 K 线、不看 IV，常常会把保险买贵？ 5. 如果你觉得纯 Protective Put 太贵，Collar 为什么会成为一个自然的降成本替代？

本章一句话总结

Protective Put 真正提供的，不是更高收益，而是让你在继续持有好仓位的同时，终于可以不靠祈祷来面对下跌。

下一章预告：Covered Call 深化应用——收租、止盈、降成本与弱对冲

如果说 Protective Put 是给持仓加保险， 下一章 Covered Call 则是在做另一种改造：

  • 我愿意让掉一部分上涨；
  • 去换现金流、退出纪律，或者一点点成本缓冲。

关键不在结构难，而在你到底想让它替你完成什么任务。
